Capitolo 01

Selettori
CSS

I selettori sono il cuore del CSS — definiscono quali elementi vengono stilizzati. Da quelli semplici per tag e classi fino ai selettori relazionali di CSS Level 4, conoscerli tutti cambia come scrivi stili.

Selettori base 
/* Universale */
* { box-sizing: border-box; }

/* Tipo */
h1, h2, h3 { font-family: sans-serif; }

/* Classe */
.card { padding: 2rem; }
.card.featured { border: 2px solid gold; }  /* più classi */

/* ID */
#header { position: sticky; top: 0; }

/* Attributo */
a[href] { color: blue; }
a[href^="https"] { color: green; }   /* inizia con */
a[href$=".pdf"] { color: red; }     /* finisce con */
a[href*="vortex"] { color: lime; }  /* contiene */
input[type="email"] { border: 1px solid #ccc; }
[data-tema="dark"] { background: #000; }
Selettori combinatori 
/* Discendente (spazio) */
nav a { text-decoration: none; }

/* Figlio diretto (>) */
ul > li { list-style: none; }

/* Fratello adiacente (+) */
h2 + p { margin-top: 0; }

/* Fratelli generici (~) */
h2 ~ p { color: gray; }

/* :is() — raggruppamento semplificato (CSS Level 4) */
:is(h1, h2, h3) span { color: lime; }

/* :where() — come :is() ma specificità zero */
:where(article, section) p { line-height: 1.8; }

/* :not() — esclude elementi */
li:not(:last-child) { border-bottom: 1px solid #eee; }
input:not([type="submit"]) { border: 1px solid #ccc; }

/* :has() — selettore genitore (CSS Level 4) */
figure:has(figcaption) { margin-bottom: 2rem; }
label:has(+ input:required) { font-weight: bold; }
div:has(> img) { padding: 0; }
💡 :has() è rivoluzionario

Il selettore :has() è il primo vero "selettore genitore" nella storia di CSS. Disponibile in tutti i browser moderni dal 2023, permette di stilizzare un elemento in base ai suoi figli o fratelli — una cosa impossibile prima senza JavaScript.

Capitolo 02

Specificità
& Cascata

La cascata CSS decide quale regola vince quando più regole si applicano allo stesso elemento. Capire specificità, ordine e inheritance è fondamentale per scrivere CSS predicibile e manutenibile.

Come si calcola

La specificità è un punteggio a tre valori: (ID, Classe/Attributo/Pseudo-classe, Tipo/Pseudo-elemento). Vince il selettore con il punteggio più alto. In caso di parità, vince l'ultimo dichiarato.

0,0,1
p { } Tipo
0,1,0
.card { } Classe
0,1,1
div.card { } Tipo + Classe
0,2,1
.nav .card { } Due classi
1,0,0
#header { } ID
1,1,0
#header .logo { } ID + Classe
∞
color: red !important !important — evitalo
✓ Specificità gestita bene
/* Solo classi, facile da sovrascrivere */
.btn { padding: 8px 16px; }
.btn-primary { background: blue; }
.btn-lg { padding: 12px 24px; }

/* layer per controllo esplicito */
@layer base, components, utilities;
✗ Specificità fuori controllo
/* ID + classe = duro da battere */
#app div.card span {
  color: red;
}
/* e ora userai !important... */
.highlight {
  color: blue !important;
}
ℹ️ @layer — controllo esplicito della cascata

Con @layer puoi dichiarare esplicitamente l'ordine di priorità dei tuoi stili. Le regole nei layer successivi vincono su quelli precedenti, indipendentemente dalla specificità. Introdotto nel 2022, supportato da tutti i browser moderni.

Capitolo 03

Box
Model

Ogni elemento HTML è una scatola rettangolare composta da content, padding, border e margin. Capire il box model — e la differenza tra content-box e border-box — è il prerequisito di ogni layout.

Box Model 
/* border-box: width e height includono padding e border */
*, *::before, *::after {
  box-sizing: border-box;  /* usa sempre questo */
}

.box {
  /* Dimensioni */
  width: 300px;
  height: 200px;
  min-width: 100px;
  max-width: 100%;
  min-height: 80px;

  /* Padding — interno */
  padding: 1rem;                      /* tutti i lati */
  padding: 1rem 2rem;               /* top/bottom left/right */
  padding: 10px 20px 30px 40px;    /* top right bottom left */
  padding-inline: 2rem;             /* logico: sinistra + destra */
  padding-block: 1rem;              /* logico: sopra + sotto */

  /* Margin — esterno */
  margin: 0 auto;                   /* centra orizzontalmente */
  margin-block-start: 2rem;        /* proprietà logica */

  /* Border */
  border: 2px solid #C6FF33;
  border-radius: 8px;
  border-radius: 50%;              /* cerchio perfetto */
  border-radius: 16px 4px 16px 4px;

  /* Overflow */
  overflow: hidden;
  overflow-x: auto;
  overflow-y: scroll;
  overflow: clip;                   /* non crea scroll context */
}
Capitolo 04

Tipo
grafia

La tipografia è il 95% del web design. Font variabili, fluid type con clamp(), proprietà di spaziatura avanzate, text-wrap: balance — il CSS moderno ha trasformato il controllo tipografico.

Tipografia completa 
/* Font loading ottimizzato */
@font-face {
  font-family: 'MioFont';
  src: url('font.woff2') format('woff2');
  font-display: swap;
  font-weight: 100 900;  /* font variabile */
}

body {
  font-family: 'Syne', system-ui, sans-serif;
  font-size: clamp(1rem, 2.5vw, 1.25rem); /* fluid */
  line-height: 1.7;
  font-weight: 400;
  letter-spacing: 0.01em;
}

h1 {
  /* Fluid typography con clamp() */
  font-size: clamp(2.5rem, 8vw, 6rem);
  line-height: 1.05;
  letter-spacing: -0.03em;
  text-wrap: balance;   /* equilibra righe */
}

p {
  text-wrap: pretty;    /* evita righe orfane */
  max-width: 65ch;       /* ~65 caratteri = leggibilità ottimale */
  hanging-punctuation: first last;
}

/* Truncation */
.truncate {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.line-clamp-3 {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}

/* Font variabili — axes */
.variable-font {
  font-variation-settings:
    'wght' 750,   /* weight */
    'wdth' 85,    /* width */
    'ital' 0.5;   /* italic */
}
Capitolo 05

Colori &
Gradienti

CSS Level 4 ha rivoluzionato il colore: oklch, color-mix(), color-contrast(), spazi colore wide-gamut, e gradienti con interpolazione nei nuovi spazi. Una tavolozza infinitamente più ricca.

Sintassi colori — tutte le forme 
/* Classici */
color: #C6FF33;
color: rgb(198, 255, 51);
color: hsl(77deg 100% 60%);
color: rgb(198 255 51 / 0.8);   /* con alpha */

/* CSS Level 4 — spazi colore moderni */
color: oklch(90% 0.23 130);     /* percezione uniforme */
color: oklch(60% 0.18 270 / 0.5); /* con alpha */
color: display-p3(0.78 1.0 0.2); /* wide gamut */

/* color-mix() — mescola colori */
color: color-mix(in oklch, lime 40%, white);
background: color-mix(in hsl, #C6FF33, transparent 30%);

/* Gradienti */
background: linear-gradient(135deg, #000, #C6FF33);
background: radial-gradient(ellipse at 30% 50%, #C6FF33, transparent);
background: conic-gradient(from 45deg, red, blue, red);

/* Gradiente con oklch — interpolazione smooth */
background: linear-gradient(in oklch, blue, red);

/* Layered backgrounds */
background:
  radial-gradient(circle at 80% 20%, oklch(85% 0.2 130 / .15), transparent 50%),
  radial-gradient(circle at 20% 80%, oklch(70% 0.15 270 / .1), transparent 50%),
  #000;

Capitolo 06

Display &
Positioning

La proprietà display controlla come un elemento partecipa al flusso del documento. Position determina come viene posizionato rispetto al suo contesto. Insieme definiscono il comportamento fondamentale di ogni elemento.

Display & Position 
/* Valori di display essenziali */
display: block;         /* occupa tutta la larghezza */
display: inline;        /* scorre nel testo, no width/height */
display: inline-block;  /* inline ma accetta dimensioni */
display: none;          /* rimuove dal flusso e dalla a11y */
display: contents;      /* rimuove il box, mantiene i figli */
display: flex;
display: grid;
display: table;

/* Visibility vs display */
visibility: hidden;   /* nascosto ma occupa spazio */
opacity: 0;           /* trasparente ma interagibile */
display: none;        /* rimosso completamente */

/* Position */
.relativo {
  position: relative;  /* offset da posizione normale */
  top: 10px; left: 20px;
}

.assoluto {
  position: absolute;  /* rispetto al genitore positioned */
  top: 0; right: 0;
  /* centra nell'ancestor */
  inset: 0; margin: auto;
}

.fisso {
  position: fixed;    /* rispetto al viewport */
  bottom: 2rem; right: 2rem;
  z-index: 1000;
}

.sticky {
  position: sticky;  /* rimane nel flusso + si ancora */
  top: 64px;         /* OBBLIGATORIO per sticky */
}

/* inset — shorthand per top/right/bottom/left */
inset: 0;                    /* tutti e 4 = 0 */
inset: 10px 20px;           /* top/bottom + left/right */
inset-inline: 0;             /* logico: left + right */
Capitolo 07

Flex
box

Flexbox è il sistema di layout monodimensionale per eccellenza — distribuisce spazio lungo un asse (riga o colonna) con un controllo finissimo su allineamento, ordine e flessibilità degli elementi.

Flexbox — riferimento completo 
/* Container */
.container {
  display: flex;

  /* Direzione */
  flex-direction: row;             /* default: sinistra → destra */
  flex-direction: row-reverse;
  flex-direction: column;
  flex-direction: column-reverse;

  /* A capo */
  flex-wrap: nowrap;              /* default */
  flex-wrap: wrap;
  flex-flow: row wrap;           /* shorthand */

  /* Spaziatura */
  gap: 1rem;
  gap: 2rem 1rem;               /* row-gap + column-gap */

  /* Allineamento asse principale (justify) */
  justify-content: flex-start;   /* default */
  justify-content: flex-end;
  justify-content: center;
  justify-content: space-between;
  justify-content: space-around;
  justify-content: space-evenly;

  /* Allineamento asse trasversale (align) */
  align-items: stretch;          /* default */
  align-items: flex-start;
  align-items: flex-end;
  align-items: center;
  align-items: baseline;
}

/* Items */
.item {
  flex: 1;                       /* flex-grow: 1, occupa spazio disponibile */
  flex: 0 0 200px;              /* grow shrink basis */
  flex: none;                    /* 0 0 auto — dimensione fissa */
  align-self: flex-end;         /* override align-items */
  order: -1;                     /* cambia ordine visivo */
}

/* Centrare qualsiasi cosa */
.centra {
  display: flex;
  align-items: center;
  justify-content: center;
}

Capitolo 08

CSS
Grid

CSS Grid è il primo sistema di layout bidimensionale nativo del web. Righe e colonne, template, named areas, auto-placement, subgrid — è il futuro (e il presente) del layout CSS.

CSS Grid — completo 
.grid {
  display: grid;

  /* Definire colonne */
  grid-template-columns: 200px 1fr 1fr;
  grid-template-columns: repeat(3, 1fr);
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  grid-template-columns: subgrid;    /* eredita la griglia genitore */

  /* Righe */
  grid-template-rows: auto 1fr auto;
  grid-auto-rows: minmax(100px, auto);

  /* Gap */
  gap: 1rem;
  column-gap: 2rem;

  /* Named areas */
  grid-template-areas:
    "header header header"
    "sidebar main    main"
    "footer footer  footer";
}

/* Posizionamento items */
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }

/* Span e linee */
.hero {
  grid-column: 1 / -1;       /* dall'inizio alla fine */
  grid-column: span 2;        /* occupa 2 colonne */
  grid-row: 1 / 3;            /* dalla riga 1 alla 3 */
}

/* Allineamento */
.grid {
  justify-items: start;    /* items nelle celle, asse X */
  align-items: center;    /* items nelle celle, asse Y */
  place-items: center;    /* shorthand per entrambi */
  justify-content: space-between; /* griglia nel container */
}

Capitolo 09

Custom
Properties

Le variabili CSS (custom properties) sono vere variabili native del browser — ereditabili, modificabili a runtime con JavaScript, scopate localmente. Sono la base di ogni sistema di design moderno.

Custom Properties — pattern avanzati 
/* Definizione nel root */
:root {
  /* Design tokens */
  --color-accent: #C6FF33;
  --color-bg: #000;
  --color-text: #fff;

  /* Spacing scale */
  --space-1: 0.25rem;
  --space-4: 1rem;
  --space-8: 2rem;
  --space-16: 4rem;

  /* Typography scale */
  --text-sm: clamp(0.8rem, 1.5vw, 0.9rem);
  --text-base: clamp(1rem, 2vw, 1.1rem);
  --text-xl: clamp(1.5rem, 4vw, 2rem);

  /* Easing */
  --ease-out: cubic-bezier(0.22, 1, 0.36, 1);
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);

  /* Shadow */
  --shadow-sm: 0 1px 3px rgb(0 0 0 / .15);
  --shadow-lg: 0 20px 40px rgb(0 0 0 / .4);
}

/* Uso */
.card {
  background: var(--color-bg);
  color: var(--color-text);
  padding: var(--space-8);
  box-shadow: var(--shadow-lg);
  /* fallback */
  border: 1px solid var(--border-color, rgba(255,255,255,.2));
}

/* Scope locale */
.btn {
  --btn-bg: transparent;
  --btn-color: var(--color-accent);
  background: var(--btn-bg);
  color: var(--btn-color);
}
.btn-filled {
  --btn-bg: var(--color-accent);
  --btn-color: #000;
}

Capitolo 10

Pseudo-classi
& Elementi

Le pseudo-classi selezionano elementi in base al loro stato o posizione. Gli pseudo-elementi creano parti virtuali di un elemento che puoi stilizzare. Insieme coprono il 90% degli effetti interattivi.

Pseudo-classi di stato

a:link    { color: blue; }
a:visited { color: purple; }
a:hover   { color: lime; }
a:active  { color: red; }
a:focus   { outline: 2px solid lime; }
a:focus-visible { outline: 2px solid lime; }

input:disabled    { opacity: .5; }
input:checked     { accent-color: lime; }
input:required    { border-color: orange; }
input:valid       { border-color: green; }
input:invalid     { border-color: red; }
input:placeholder-shown {
  border-color: gray;
}

Pseudo-classi strutturali

li:first-child    {}
li:last-child     {}
li:nth-child(2)   {}  /* 2° figlio */
li:nth-child(odd) {}  /* dispari */
li:nth-child(3n)  {}  /* ogni 3 */
li:nth-child(n+4) {}  /* dal 4° in poi */
li:nth-last-child(2) {}

/* Nuovo in CSS Level 4 */
li:nth-child(2 of .active) {}

p:first-of-type {}
p:last-of-type  {}
p:only-child    {}
p:empty         {}
:root           {}  /* elemento html */

Pseudo-elementi

Pseudo-elementi 
/* ::before e ::after — contenuto generato */
.card::before {
  content: '';              /* obbligatorio */
  display: block;
  width: 3px;
  height: 100%;
  background: lime;
  position: absolute;
  left: 0;
}

/* ::after con content dinamico da data-attr */
[data-tooltip]::after {
  content: attr(data-tooltip);
}

/* ::placeholder — stile del placeholder */
input::placeholder {
  color: rgba(255,255,255,.3);
  font-style: italic;
}

/* ::selection — testo selezionato */
::selection {
  background: #C6FF33;
  color: #000;
}

/* ::marker — punto delle liste */
li::marker {
  color: lime;
  font-size: 1.2em;
}

/* ::first-line, ::first-letter */
p::first-letter {
  font-size: 3em;
  float: left;
  line-height: 0.8;

Capitolo 11

Transizioni
& Animazioni

Le animazioni CSS sono performanti perché vengono gestite dal compositor del browser sul GPU — senza toccare il main thread. Transition per stati, @keyframes per sequenze, animation-timeline per lo scroll.

Transizioni & @keyframes completo 
/* Transition — da stato A a stato B */
.btn {
  transition: all 0.3s ease;
  transition:
    color 0.2s ease,
    background 0.2s ease,
    transform 0.3s cubic-bezier(0.34,1.56,0.64,1);
}
.btn:hover {
  transform: translateY(-4px) scale(1.02);
}

/* Proprietà animabili performanti */
/* ✓ GPU: transform, opacity, filter */
/* ✗ CPU: width, height, margin, padding, top, left */

/* @keyframes */
@keyframes slideIn {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes shimmer {
  0%   { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}

/* Applicare animazione */
.elemento {
  animation:
    slideIn 0.6s cubic-bezier(.22,1,.36,1) both;
  animation-delay: 0.2s;
  animation-fill-mode: both;
}

/* Scroll-driven animations (CSS 2023) */
.progress {
  animation: grow-bar linear;
  animation-timeline: scroll();
}

/* Rispetta preferenze utente */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

Capitolo 12

Responsive
Design

Il responsive design non è solo media query. È un approccio fluido che usa clamp(), unità relative, auto-fit, container queries e la logica mobile-first per creare interfacce che funzionano su qualsiasi schermo.

Media & Container Queries 
/* Mobile-first (aggiungi complessità al crescere) */
.grid {
  display: grid;
  grid-template-columns: 1fr;    /* mobile: 1 colonna */
  gap: 1rem;
}

@media (min-width: 640px) {
  .grid { grid-template-columns: repeat(2,1fr); }
}

@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3,1fr); }
}

/* Range syntax (moderno) */
@media (600px <= width < 1200px) {
  /* solo tablet */
}

/* Auto-responsive senza media query */
.cards {
  display: grid;
  grid-template-columns:
    repeat(auto-fit, minmax(min(280px, 100%), 1fr));
}

/* Container Queries (2023) — responsive al container */
.sidebar {
  container-type: inline-size;
  container-name: sidebar;
}
@container sidebar (min-width: 400px) {
  .card { flex-direction: row; }
}

/* Unità viewport moderne */
height: 100dvh;   /* dynamic viewport height */
height: 100svh;   /* small viewport height */
height: 100lvh;   /* large viewport height */

/* Preferenze sistema */
@media (prefers-color-scheme: dark) { }
@media (prefers-reduced-motion: reduce) { }
@media (prefers-contrast: high) { }
@media (hover: none) { }       /* touch device */
@media (pointer: coarse) { }   /* touch pointer */
Capitolo 13

Funzioni
CSS

CSS ha un sistema di funzioni sempre più ricco. clamp(), min(), max(), calc(), env(), round() e le nuove funzioni matematiche di CSS Level 4 portano logica computazionale direttamente negli stili.

Funzioni CSS essenziali e avanzate 
/* calc() — calcoli con unità miste */
width: calc(100% - 240px);
margin-top: calc(var(--spacing) * 2);
font-size: calc(1rem + 0.5vw);

/* min() / max() / clamp() */
width: min(600px, 90%);      /* il più piccolo */
width: max(300px, 50%);      /* il più grande */
font-size: clamp(1rem, 2.5vw, 2rem);
//                 min   fluid   max

/* Nuove funzioni matematiche CSS Level 4 */
width: round(nearest, 1.5rem, 0.5rem);  /* 1.5 */
width: mod(18px, 5px);    /* 3px */
transform: rotate(atan2(1, 2));
width: abs(-10px);        /* 10px */
padding: sign(-5) * 1rem; /* -1rem */

/* env() — variabili dell'ambiente (Safe Area iOS) */
padding-bottom: env(safe-area-inset-bottom);
padding-top: max(1rem, env(safe-area-inset-top));

/* transform functions */
transform:
  translateX(100px)
  translateY(-50%)
  rotate(45deg)
  scale(1.1)
  skewX(10deg)
  perspective(500px)
  rotateY(30deg);
Capitolo 14

CSS
Moderno

@layer, @scope, nesting nativo, cascade layers, logical properties, view transitions, anchor positioning — il CSS del 2023–2025 ha aggiunto più funzionalità di qualsiasi altro periodo della sua storia.

CSS Nesting nativo (2023) 
/* Prima: Sass/Less. Ora: CSS nativo */
.card {
  padding: 2rem;
  background: #111;

  /* Nesting — riferisce al genitore con & */
  &:hover {
    background: #1a1a1a;
  }

  & .title {
    font-size: 1.5rem;
  }

  & .title:hover {
    color: lime;
  }

  /* Media query annidata */
  @media (min-width: 768px) {
    padding: 3rem;
  }
}

Capitolo 15

Best
Practice

CSS scalabile non è solo CSS che funziona — è CSS che rimane manutenibile, performante e leggibile a distanza di mesi. Metodologie, performance, accessibilità e gli errori comuni da evitare.

Scrittura & Organizzazione

  • Usa sempre box-sizing: border-box globalmente
  • Mobile-first: scrivi il base e aggiungi con min-width
  • Preferisci classi agli ID per gli stili
  • Evita selettori troppo profondi (max 3 livelli)
  • Usa Custom Properties per tutti i valori ripetuti
  • Organizza con @layer: base, components, utilities
  • Usa CSS Nesting per tenere gli stili vicini al componente
  • Commenta le sezioni e le scelte non ovvie

Performance

  • Anima solo transform e opacity (GPU)
  • Usa will-change solo se necessario e rimuovilo dopo
  • Carica i font con font-display: swap
  • Usa preconnect per Google Fonts e CDN
  • Evita CSS critici non usati — purga in produzione
  • Usa contain: layout per isolare i layout
  • Preferisci content-visibility: auto per sezioni lunghe
  • Minifica il CSS in produzione

Accessibilità

  • Non rimuovere mai :focus senza alternativa visibile
  • Usa :focus-visible per focus solo da tastiera
  • Contrasto minimo 4.5:1 per testo normale
  • Rispetta prefers-reduced-motion
  • Rispetta prefers-color-scheme
  • Non usare solo il colore per trasmettere informazioni
  • Evita display:none su skip links (usa sr-only)
  • Test con zoom al 200% senza perdita di funzionalità

Errori comuni da evitare

  • Mai usare !important — risolvi la specificità
  • Mai usare margin: 0 auto su flexbox/grid items
  • Non animare proprietà che causano reflow (width, height)
  • Non usare px per font-size (usa rem)
  • Non bloccare lo scroll nativo con overflow inutili
  • Non usare * per reset pesanti in produzione
  • Non dimenticare il fallback nei gradient con oklch
  • Non usare z-index con valori enormi (999999)
Fine Guida
🚀 Prossimi passi

Con CSS solido padroneggiato, esplora i preprocessori come Sass per funzionalità extra, oppure i framework utility-first come Tailwind CSS. Per applicazioni complesse studia i CSS-in-JS e le metodologie BEM, CUBE CSS e ITCSS. La trilogia è completa: HTML struttura, CSS stile, JavaScript comportamento.
